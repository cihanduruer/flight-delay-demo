# Documentation index

This folder contains product, data, and decision documentation for the flight delay probability application.

## How to use these docs

- Start with the problem statement and scope in `product/overview.md`.
- Review decisions in `adr/` (Architectural Decision Records).
- Review the planned milestones in `plan/`.
- Review detailed user interactions in `../use-cases/`.

## Conventions

- Markdown files are written to be implementation-agnostic.
- Decisions are recorded as ADRs so they can evolve without losing history.
- Use cases are one-file-per-use-case so new interactions can be added cleanly.
